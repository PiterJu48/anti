엑셀 파일의 서식, 디자인, 스타일(배경색, 테두리, 셀 병합 등)을 완벽하게 보존하면서 데이터만 안전하게 업데이트하기 위해 AI에게 프롬프트를 작성해 전달할 때의 **가장 효과적인 방식과 가이드라인**을 안내해 드립니다.

### 1. 프롬프트 작성 시 가장 효과적인 방법: **"하이브리드 지정 방식"**

결론부터 말씀드리면, **"농장 정보와 같은 고정 영역은 구체적인 셀 주소(좌표)를 직접 지정"**하고, **"평가 항목 결과값처럼 행 수가 많고 가변적인 영역은 '텍스트 매칭 규칙'을 지정"**해 주는 하이브리드 방식이 가장 좋습니다.

*   **고정 정보 (농장 정보 등)**: `A3`, `B3`, `C3` 등 셀 좌표를 프롬프트에 직접 작성해 주는 것이 가장 명확하고 오류가 없습니다.
*   **평가 결과 정보 (G열 등)**: 문항 수가 많고 추후 평가표가 수정되어 행 위치가 위아래로 밀릴 수 있기 때문에, 하드코딩 좌표(`G7`, `G8` 등)를 일일이 지정하기보다 **"B열(혹은 C열)에 적혀 있는 평가 문항 텍스트와 매칭되는 행의 G열에 업데이트하라"**는 규칙(텍스트 매칭)을 주는 것이 훨씬 안전하고 유지보수가 쉽습니다.

### 2. AI에게 전달하기 가장 좋은 프롬프트 템플릿 예시

향후 다른 AI나 개발 도구에 이 작업을 자동으로 해달라고 요청할 때 아래 프롬프트를 복사하여 전달하시면 가장 정확한 코드를 얻으실 수 있습니다.

```markdown
[요청 사항]
제공된 엑셀 파일(result.xlsx)의 기존 스타일, 글꼴, 셀 병합, 서식 디자인을 절대 손상시키지 않고, 특정 셀의 값만 동적으로 업데이트하여 다시 저장하는 JavaScript(또는 Python) 코드를 작성해 줘.

[데이터 매핑 규칙]
1. 기존 엑셀 파일(result.xlsx)을 템플릿으로 읽어와 수정합니다.
2. 농장 기본 정보는 다음 고정 셀 좌표에 업데이트합니다.
   - (A, 3) 셀 : 농장명
   - (B, 3) 셀 : 대표자명 (농장주)
   - (C, 3) 셀 : 주소
   - (E, 3) 셀 : 시설규모
   - (F, 3) 셀 : 전화번호
   - (G, 3) 셀 : 종합 의견 (비고)

3. 평가 결과 매핑 (동적 매칭 규칙):
   - 6행부터 137행까지 순회하며, 각 행의 B열(또는 C열)에 적힌 '평가 기준' 텍스트를 검사합니다.
   - 입력 데이터(DB 또는 JSON)의 평가 문항 기준 텍스트와 가장 유사하거나 일치하는 행을 찾습니다.
   - 매칭된 행의 'G열' (7번째 열)에만 평가 결과값(Y, N 또는 점수)을 덮어씁니다.
   - 이 과정에서 엑셀 서식 파일의 스타일 속성(s 속성)이나 병합 구조가 깨지지 않도록 셀의 `.v` (값) 속성만 안전하게 변경해야 합니다.

4. 서명 및 일시 영역:
   - (A, 122) 셀 : "인증심사기간 : YYYY년 MM월 DD일" 형식으로 기재
   - (A, 123) 셀 : "인증심사원 : {이름}" 형식으로 기재
```

### 3. 기술적으로 안전하게 보존하기 위한 주의 사항 (개발 시 팁)
*   **값만 변경 (`.v` 속성)**: SheetJS(xlsx)나 openpyxl 같은 라이브러리를 쓸 때 셀 전체 객체를 새로 덮어쓰면 테두리나 색상 정보가 완전히 지워집니다. 반드시 `ws['A3'].v = '값'` 처럼 값만 대입하도록 프롬프트에 명시해 주어야 합니다.
*   **캐시 방지**: 브라우저 환경에서 이 작업을 돌릴 때는 템플릿 파일을 fetch 할 때 브라우저가 옛날 버전을 캐싱하여 빈 문서가 다운로드되는 현상이 자주 발생하므로, 파일 경로 뒤에 캐시 버스팅(`?_ts=Date.now()`) 처리를 꼭 해주어야 템플릿 손상 오류를 방지할 수 있습니다.
